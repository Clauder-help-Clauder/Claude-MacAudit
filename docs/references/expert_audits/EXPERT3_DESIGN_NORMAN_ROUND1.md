# Expert 3: Don Norman — 产品哲学与设计心理学审查 Round 1

## 审查焦点
**Affordances（示能）与 Signifiers（意符）** — UI元素是否自然暗示其操作方式？用户能否看到操作提示？

## 发现的问题

### 问题 1: Module Strip 卡片缺乏可点击的意符
- **严重程度**: HIGH
- **设计原则违反**: Signifier（意符）
- **文件**: `DashboardView.swift:334-378`
- **描述**: moduleCard 被包裹在 `Button` 中，但没有任何视觉意符暗示它是可点击的——没有 hover 效果、没有 chevron、没有指针变化提示。用户看到的是一个信息展示卡片（模块名+分数+通过数），而非导航按钮。
- **用户心理影响**: 困惑——"这些卡片能点吗？" 用户可能永远不点击，导致发现路径断裂。
- **修复建议**: 添加 hover 状态（如边框亮起或微妙的背景色变化），或在右侧添加 `chevron.right` 图标作为导航意符。

### 问题 2: SpectralButton ghost 风格的 affordance 模糊
- **严重程度**: MEDIUM
- **设计原则违反**: Affordance（示能）+ Signifier（意符）
- **文件**: `Components.swift:165-214`
- **描述**: ghost 按钮使用 `[ INITIATE FULL SYSTEM AUDIT ]` 的括号装饰风格。括号是装饰性元素，不是按钮的通用意符。虽然 hover 时边框变亮提供了一些反馈，但静态状态下，它与纯文本标签的视觉区分度极低。macOS 用户习惯于圆角矩形按钮或填充色按钮。
- **用户心理影响**: 轻微迟疑——"这是按钮还是装饰性标题？" 需要鼠标悬停才能确认。
- **修复建议**: 保持当前风格的同时增加微妙的填充色（如极淡的绿色背景），或添加一个小的播放/扫描图标作为操作意符。

### 问题 3: "LAST RESULTS" 按钮的视觉层级过低
- **严重程度**: MEDIUM
- **设计原则违反**: Signifier（意符）+ Mapping（映射）
- **文件**: `DashboardView.swift:112-134`
- **描述**: 上次结果入口使用了极低对比度的颜色（`#5A6B52`、`#3C4B35`），在深色背景上几乎不可见。这是用户继续上次工作的关键入口，但视觉权重低于底部装饰性文字"QUANTUM-LEVEL HEURISTIC SCAN READY"。
- **用户心理影响**: 遗漏——用户可能完全忽略这个入口，被迫每次重新扫描。
- **修复建议**: 提高文字对比度至 `.textMuted` 级别，添加一个小的时钟/历史图标作为意符。

### 问题 4: COPY 按钮与 SKIP 按钮缺乏操作风险层级的视觉区分
- **严重程度**: MEDIUM
- **设计原则违反**: Mapping（映射）
- **文件**: `ResultsView.swift:139-166`（COPY）, `ResultsView.swift:411-424`（SKIP）
- **描述**: COPY 按钮使用 `.statusFail` 红色系，SKIP 按钮使用暗灰色。但 COPY 是安全操作（仅复制文本），SKIP 才是改变系统状态的决策（跳过安全检查）。红色被用于安全操作，灰色被用于决策操作，颜色映射与操作风险不匹配。
- **用户心理影响**: 误导——红色通常暗示危险，但这里用于安全的复制操作。
- **修复建议**: COPY 应使用中性色（如 `.linkCyan`），SKIP 应使用 `.statusWarn` 黄色暗示"注意：你在忽略一个问题"。

### 问题 5: CheckRow 仅 fail 项可点击但缺少明确的可点击意符
- **严重程度**: LOW
- **设计原则违反**: Signifier（意符）
- **文件**: `ResultsView.swift:700-768`
- **描述**: `isTappable` 逻辑仅对 `.fail` 状态显示 chevron 和加粗文字。但 `pass` 和 `warn` 项同样被 `onTapGesture` 包裹（在 CheckListView 中），用户点击后能跳转详情——但视觉上没有意符暗示这些项是可点击的。
- **用户心理影响**: 发现障碍——用户可能只点击 FAIL 项，错过 WARN/PASS 的详情信息。
- **修复建议**: 对所有可点击项统一添加 hover 效果，或对非 FAIL 项添加更轻量的 chevron。

## Tacit Knowledge 审查

1. **"赛博朋克"美学 vs 可用性的张力**：整体视觉语言（Matrix 下落点、幽灵网格、暗黑主题）追求强烈的审美表达，但装饰性元素（旋转虚线环、呼吸指示灯）可能与功能性 UI 竞争注意力。用户在"这是什么？"和"我该做什么？"之间会产生微妙的认知冲突。

2. **RingChart 的双重编码**：分数 + 颜色 + 标签（OPTIMAL DEFENSE 等）提供了三层信息编码，这是好的。但"SYSTEM SCORE"文字标签与用户心智模型中"安全评分"的映射可能不直觉——用户更习惯"Security Score"或"Protection Level"。

3. **视觉噪音阈值**：Dashboard 空状态同时存在 6 种动态元素（旋转装饰圈、呼吸脉冲灯、下落点、Matrix 背景点、网格背景、光晕球体）。虽然单个都很微妙，但累积效应可能让用户感到"屏幕在动但我不确定该做什么"。
